## [→ ТВОЙ ХОД] Когда приедут ноды

Нужен USB-UART адаптер на **3.3 В** (не 5 В — сожжёшь) и провода.

### Шаг 1. Прошить

**V4 (ESP32-S3) — кликом:**

1. Открой https://hleserg.github.io/meshcore-node-fw/ **в Chrome или Edge** (в Firefox
   и Safari кнопка физически не заработает — там нет Web Serial).
2. Воткни V4 в USB.
3. Вкладка «Heltec WiFi LoRa 32 V4» → кнопка **«Прошить V4 через браузер»**.
4. Браузер покажет список портов — выбери свой.

*Как выглядит успех:* прогресс-бар до 100%, потом «Installation complete».

**T114 (nRF52840) — файлом:**

1. На странице вкладка «Heltec Mesh Node T114» → **«Скачать t114-companion.uf2»**.
2. Воткни T114 в USB.
3. **Быстро нажми `RST` два раза.**
4. Появится диск `T114BOOT` — перетащи на него скачанный `.uf2`.

*Как выглядит успех:* диск сам исчезает, плата перезагружается.

### Шаг 2. Глянуть стартовый баннер (сверка пинов)

Не отключая USB, открой USB-порт платы **терминалом на 115200** и нажми `RST`.

Должно выпасть:

```
=== MeshCore companion :: hardware UART ===
firmware  : v0.1.0-… (…)
port      : Serial2 (NRF_UARTE)      <- на V4 будет "UART2"
RX        : arduino 9 -> P0.09       <- на V4 будет "GPIO47"
TX        : arduino 10 -> P0.10      <- на V4 будет "GPIO48"
baud      : 115200 8N1
```

**Это и есть сверка «силк ≠ Arduino-пин».** Если тут `P0.09`/`P0.10` — распиновка та,
что заявлена, и можно паять. Если что-то другое — стоп, напиши мне, дальше не иди.

### Шаг 3. Подключить провода

| T114 (хедер P1) | | V4 (хедер J3) | |
|---|---|---|---|
| `GPIO9` (P0.09) | ← **TX** адаптера | `GPIO47` | ← **TX** адаптера |
| `GPIO10` (P0.10) | → **RX** адаптера | `GPIO48` | → **RX** адаптера |
| `GND` | ↔ GND | `GND` | ↔ GND |

RX платы идёт на TX адаптера. Крест-накрест. Земля обязательна.

### Шаг 4. Проверить self-info

```bash
pip install meshcore
python scripts/selftest.py COM5          # порт АДАПТЕРА, не платы
python scripts/selftest.py /dev/ttyUSB0  # Linux / macOS
```

*Как выглядит успех:* печатается self-info — публичный ключ, версия прошивки, параметры
радио, и в конце `OK - the companion protocol is alive on the hardware UART.`

Версия в ответе должна совпасть с той, что была в баннере на шаге 2.

## Если не так → почему → что делать

| Симптом | Причина | Фикс |
|---|---|---|
| Баннер на шаге 2 показывает пины, отличные от `P0.09`/`P0.10` | Силк Heltec не совпал с Arduino-пином на твоей ревизии платы | Стоп. Пришли мне вывод баннера — поправлю `SERIAL_RX`/`SERIAL_TX` |
| Баннера нет вообще | Терминал открыт не на том порту, либо порт занят | Закрой другие терминалы/Arduino IDE. Порт **платы**, не адаптера. Нажми `RST` при открытом терминале |
| Баннер есть, а `selftest.py` молчит | 9 из 10 — перепутаны RX/TX | Поменяй местами два провода. Крест-накрест, не «RX к RX» |
| `selftest.py` молчит, провода точно верные | Нет общей земли | Соедини GND платы и GND адаптера |
| То же, земля есть | Адаптер на 5 В | Нужен 3.3 В. На 5 В плата может быть уже повреждена |
| T114: первый запуск после прошивки тупит пару секунд | Прошивка однократно переписывает UICR (освобождает NFC-пины) и ребутится | Так и задумано. Подожди 2–3 с |
| T114 после смены прошивки ведёт себя странно: не поднимается ФС, слетели настройки | Остатки предыдущей прошивки в файловой системе | Разово залей erase-`.uf2` из [MeshCore/bin](https://github.com/meshcore-dev/MeshCore/tree/main/bin), потом наш образ заново |
| На V4 кнопка «Прошить» серая / «Браузер не поддерживает» | Firefox или Safari — там нет Web Serial | Только Chrome или Edge, на десктопе |
| Кнопка есть, но порт не открывается | Страница не по HTTPS, либо порт занят | Открывай именно `https://hleserg.github.io/…`. Закрой терминалы, держащие порт |
| Прошил, но companion отвечает по USB, а не по проводу | Залита апстримная `*_companion_radio_usb`, а не наша | Качай только с нашей страницы или из [Releases](../../releases) |
| Страница отдаёт старую прошивку | Кэш браузера | `Ctrl+Shift+R`. Версию видно в подвале страницы |
| Pages не задеплоился (страница 404) | Job `Deploy flasher page` упал | `gh run list` → посмотри последний ран на `main`. Один известный случай: гонка двух деплоев, уже починена |

## Само-ревью: что я проверил, а что предположил

Прогнал по своим же файлам: `TODO`/`FIXME` — ноль, секретов и токенов — ноль, ни одного
радио- или сетевого параметра мы сами не задаём.

Но два момента честнее сказать прямо, а то «радио не захардкожено» звучит шире, чем есть:

**1. Девственная плата всё-таки поднимется на конкретной частоте.** Из апстримного
`[arduino_base]` наследуются дефолты, и мы их не отменяем:

```
-D LORA_FREQ=869.618   -D LORA_BW=62.5   -D LORA_SF=8   -D LORA_TX_POWER=22
-D ENABLE_ADVERT_ON_BOOT=1
```

На чистой файловой системе (первая прошивка либо после erase) нода стартует **именно на
869.618 МГц** и, из-за `ENABLE_ADVERT_ON_BOOT`, сразу даёт advert в эфир — до того, как
ты что-то настроишь. Дальше `loadPrefs()` перетирает это сохранённым, и параметры
становятся полностью runtime. То есть «не захардкожено» = «не залочено», но дефолт
существует и он не нейтральный. Если это не та частота, с которой ты хочешь стартовать —
скажи, вынесу в build-флаг нашего env.

**2. Экспорт приватного ключа включён.** Тоже унаследовано:

```
-D ENABLE_PRIVATE_KEY_IMPORT=1
-D ENABLE_PRIVATE_KEY_EXPORT=1
```

Апстрим рядом с ними пишет: *«comment these out for more secure firmware»*. Приватный
ключ ноды можно выгрузить командой companion-протокола. У нас companion висит на проводе,
так что нужен физический доступ к пинам — порог заметно выше, чем по BLE. Оставил как в
апстриме, но это осознанный выбор, а не недосмотр. Скажи — выключу.

**Чего я не делал вообще:** не трогал живое железо. Всё «зелёное» выше — это компиляция,
статический разбор бинарей и HTTP-проверки сайта. Рукопожатие по UART не проверено
никем.

## Что стоит решить

- **Страница отдаёт сборку с вершины `main`, а не релизную.** Сегодня это одно и то же
  (тот же коммит сабмодуля), но дальше стоит выбрать: флешер следует за релизами или за
  `main`. Я по умолчанию оставил `main` — свежее.
- **Профильные варианты прошивок не делались** и, по-моему, не нужны: compile-time
  сетевых параметров в MeshCore нет, обоснование в README. Если знаешь контрпример —
  скажи, переделаю.
