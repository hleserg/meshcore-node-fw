# Фикстура для проверки рендера гайдов

Этот файл не попадает на сайт. Он существует, чтобы `render_guide.py` можно было
прогнать по всем конструкциям, которые встречаются в боевых инструкциях, и убедиться,
что ничего не теряется.

## Часть 1. Подготовка

Обычный абзац с **жирным**, *курсивом*, `инлайн-кодом` и [ссылкой](https://example.com).

### Подраздел с командой

Блок кода — у него обязана появиться copy-кнопка:

```bash
python meshlog.py --tcp 192.168.4.1:5000
```

Второй блок кода, без языка:

```
> 3E 00 12 ...
< 3C 00 04 ...
```

## Грабли

> ⚠️ Перепутанные RX/TX — самая частая причина тишины на линии.
> Поменяй два провода местами.

> 🚨 Адаптер на 5 В убивает плату. Только 3.3 В.

> 💡 Версию прошивки видно и в баннере, и в self-info — они должны совпадать.

### Таблица симптомов

| Симптом | Причина | Решение |
|---|---|---|
| Тишина на линии | RX/TX перепутаны | Поменять местами |
| Мусор в терминале | Не та скорость | Выставить 115200 |
| Плата не видна | Нет общей земли | Соединить GND |

## Словарик

**advert** — объявление ноды в эфир.

**companion** — режим, в котором нода управляется внешним хостом.

**path hash** — префикс публичного ключа, по которому строится маршрут.
